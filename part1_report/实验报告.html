读取原始数据

按行读取并使用正则表达式解析field与content。

遇到的问题

由于一行不一定包含完整的field，例如：

<基金>=安徽建筑工业学院硕博科研启动项目(No.2006120122);;
国家自然科学基金(50974002、41172147);;
安徽省教育厅自然科学重点项目(KJ2010A0580)

因此在按行扫描时还加入了一些trick。

field属性的设置

所有field均有索引属性，仅以下field带有存储属性：

    void initStoredSet(){
        storedSet.add("题名");
        storedSet.add("作者");
        storedSet.add("关键词");
        storedSet.add("出版单位");
        storedSet.add("出版日期");
        storedSet.add("摘要");
        storedSet.add("专题名称");
        storedSet.add("分类名称");
    }

这些也正是搜索结果显示时一个条目的内容。

String[] allFields = {

            "语种", "引证文献", "共引文献", "二级引证文献", "表名", "出版日期", "引证文献数量", "二级参考文献数量", "二级引证文献数量",

搜索词：计算机

StandardAnalyzer

一元分词会将计算机划分为计/算/机3个字，其搜索结果的第一条中，存在设计、挤压机、计算方法单词，因此被搜索出来。

CJKAnalyzer

二元分词的搜索结果较一元分词的结果好了非常多，10条结果均与计算机相关。

AnsjAnalyzer

使用中文分词器进行分词，结果与二元分词差不多，10条结果均与计算机相关。

搜索词：机器学习

StandardAnalyzer

可以搜出人工智能和机器人，但未排在第一位，第一位为基于Matlab的农业机器的优化配备，与机器学习基本无关。

CJKAnalyzer

可以搜出人工智能和机器人，且排在第一位，搜索结果较StandardAnalyzer好。

AnsjAnalyzer

无法搜出人工智能和机器人，搜索结果大多与“学习”相关而与“机器”无关，结果相当差，甚至差于一元分词。

ansj对”机器学习“的分词结果为：机器/n,学习/v，但搜索引擎没有很好地把这两个词结合起来搜索，以至搜不出人工智能，有待优化。
